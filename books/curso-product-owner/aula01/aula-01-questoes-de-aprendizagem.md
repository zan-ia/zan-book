# Product Owner — Do Zero ao PO com Agentes Aula 01 — Questões de Aprendizagem

## Como Usar Este Arquivo

Este arquivo é o seu **checkpoint de aprendizagem**. A pergunta central é: "eu realmente entendi a matéria da Aula 01?"

Cada questão abaixo testa um conceito-chave da aula. Siga a ordem — elas progridem do mais simples para o mais complexo. Para cada questão, leia o **Objetivo**, siga os **Passos de Execução** e preencha o **Template de Entrega** na pasta `entregas-aula-01/`.

**Importante:** responda SEM consultar a aula. Se travar em alguma questão, anote em qual seção você travou — isso indica o que você precisa revisar. O gabarito NÃO está neste arquivo (ele está na aula principal). Você só avança para a Aula 02 quando conseguir completar todas as questões por conta própria.

## Questão 4: O PO Faz e Não Faz

**Conceito-chave:** O que um PO faz e NÃO faz no dia a dia (Aula 01, Seção 3).

**Objetivo:** Demonstrar que você distingue claramente as responsabilidades do PO das responsabilidades do time de desenvolvimento e de outros papéis.

**Passos de Execução:**

1. Leia as 10 atividades abaixo
2. Classifique cada uma como **Faz** (papel do PO) ou **Não Faz** (não é papel do PO)
3. Justifique cada classificação em uma frase

Atividades:

a) Entrevistar 5 usuários para entender por que eles abandonam o carrinho de compras
b) Decidir que o banco de dados será PostgreSQL em vez de MySQL
c) Priorizar 3 funcionalidades para a próxima sprint, deixando 7 de fora
d) Escrever o código da tela de login
e) Apresentar o plano do próximo trimestre para os investidores
f) Definir o horário da reunião diária do time
g) Dizer "não, essa funcionalidade não é prioridade agora"
h) Fazer a contratação de um novo programador para o time
i) Explicar para o time de tecnologia por que uma funcionalidade específica é crítica para o negócio
j) Testar manualmente se o botão de pagamento está funcionando
